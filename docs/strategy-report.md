## 一、核心能力速览

### 1.1 数据源矩阵

| 类型         | 平台                                | 采集方式                         | 数据维度                        |
| ---------- | --------------------------------- | ---------------------------- | --------------------------- |
| **热搜 API** | 微博 / 百度 / 抖音 / 知乎 / B站            | UApiPro REST API             | 标题、热度值、排名、URL               |
| **深度采集**   | 小红书 / 抖音 / 快手 / B站 / 微博 / 贴吧 / 知乎 | MediaCrawler 子进程（Playwright） | 内容正文、评论、创作者、点赞/分享/收藏数、IP 属地 |
| **联网搜索**   | Web Search（自研 + 搜狗 + 夸克）          | iotchange-sdk                | 标题、摘要、URL、发布日期              |

**总计：12 个数据源（5 热搜 + 7 深度 + 多引擎搜索）**

### 1.2 AI Agent Skills

| 技能                 | 命令                  | 一句话描述                    |
| ------------------ | ------------------- | ------------------------ |
| `spide-crawl`      | `/spide-crawl`      | 一键采集指定平台热搜榜单             |
| `spide-deep-crawl` | `/spide-deep-crawl` | 深度抓取内容 + 评论 + 创作者        |
| `spide-batch`      | `/spide-batch`      | 多平台并发采集，Semaphore 并发控制   |
| `spide-schedule`   | `/spide-schedule`   | Cron 定时采集，可配间隔和次数        |
| `spide-analyze`    | `/spide-analyze`    | 趋势分析 + 摘要 + 情感 + 采集策略    |
| `spide-export`     | `/spide-export`     | JSON / CSV / Excel 多格式导出 |
| `spide-wordcloud`  | `/spide-wordcloud`  | jieba 分词 + 词云可视化         |

## 二、市场定位分析

### 2.1 竞品格局

| 定位            | 代表产品                   | 定价    | Spide 差异化                        |
| ------------- | ---------------------- | ----- | -------------------------------- |
| **企业舆情 SaaS** | 识微商情、鹰眼速读网、拓尔思 TRS     | 年费万元级 | Spide 是**开源 CLI Agent**，零门槛，本地部署 |
| **开源聚合**      | TrendRadar、GitHubDaily | 免费    | Spide 多 **7 平台深度采集 + AI 分析**     |
| **数据采集工具**    | Bright Data、Firecrawl  | 按量付费  | Spide 面向**中文社媒热点场景**深度优化         |
| **新媒体管理**     | 新榜、云探                  | 月费千元级 | Spide 提供 **MCP 协议**，可接入任意 AI 模型  |

### 2.2 SpideHarness 核心竞争力

```
开源免费 + 12 数据源 + AI 原生 + MCP 协议 + CLI Agent
```

1. **开源免费** — 对标万元级舆情 SaaS，开发者零成本上手
2. **12 源聚合** — 覆盖微博/抖音/小红书/B站等全部主流中文平台
3. **AI 原生** — LLM 驱动的摘要/情感/趋势/策略分析，非简单关键词匹配
4. **MCP 协议** — 可接入 Claude/GPT 等任意 MCP 兼容模型
5. **Agent CLI** — 命令行即服务，可嵌入 CI/CD、定时任务、数据管道

## 三、目标用户画像分类

### 3.1 六大核心用户群

| 用户画像           | 典型场景          | 核心需求                  | 优先级    |
| -------------- | ------------- | --------------------- | ------ |
| **自媒体创作者**     | 追热点写文章/做视频    | 实时热搜聚合 + 趋势预判 + 内容灵感  | **P0** |
| **品牌营销团队**     | 品牌舆情监控 + 竞品分析 | 关键词预警 + 情感分析 + 定时报告   | **P0** |
| **数据分析师**      | 行业报告 + 舆情研究   | 批量采集 + 多格式导出 + 趋势对比   | **P0** |
| **开发者 / 极客**   | 二次开发 + 数据管道   | CLI + MCP API + 可编程接口 | **P1** |
| **公关 / 危机管理**  | 负面舆情发现 + 应急响应 | 实时监控 + 告警 + 情感预警      | **P1** |
| **投资 / 金融从业者** | 热点事件对市场影响分析   | 热搜 + 财经关联 + 趋势预判      | **P2** |

### 3.2 画像详细拆解

#### P0-1：自媒体创作者（内容驱动型）

```
身份：独立博主、短视频创作者、公众号运营
年龄：22-35 岁
工具：手机为主，偶尔用电脑
痛点：
  - 不知道今天该写什么，错过热点流量
  - 多平台热搜分散，逐个看耗时
  - 不知道哪些热点值得追，追了没流量
期望：
  - 打开即看到全网热搜聚合
  - AI 告诉我哪个热点值得追
  - 一键生成热点内容大纲
```

#### P0-2：品牌营销团队（监控驱动型）

```
身份：品牌经理、社媒运营、市场总监
团队：3-10 人
预算：有工具采购预算但不想花大钱
痛点：
  - 品牌被黑了 2 小时才发现
  - 竞品出了新动作不知道
  - 老板要舆情日报，手动整理到半夜
期望：
  - 品牌关键词实时监控 + 微信/钉钉告警
  - 自动生成日报/周报
  - 竞品动态自动追踪
```

#### P0-3：数据分析师（数据驱动型）

```
身份：数据分析师、市场研究员、咨询顾问
技能：Python + SQL + BI 工具
痛点：
  - 采集数据耗时占 80%，分析只占 20%
  - 各平台数据格式不统一
  - 无法批量历史数据对比
期望：
  - CLI 批量采集 + JSON/CSV/Excel 导出
  - 统一数据模型，跨平台对比
  - 可嵌入 Jupyter / Airflow 管道
```

#### P1-1：开发者 / 极客（集成驱动型）

```
身份：全栈工程师、AI 应用开发者、开源贡献者
技能：CLI + API + MCP 协议
痛点：
  - 舆情平台不提供 API
  - 自建爬虫维护成本高
  - 数据无法接入自己的 AI 应用
期望：
  - MCP Server 即服务，接入任意 AI 模型
  - CLI 可嵌入 CI/CD 管道
  - 开源可二次开发
```

#### P1-2：公关 / 危机管理（预警驱动型）

```
身份：PR 总监、企业公关、政府舆情岗
场景：7×24 舆情监控 + 负面预警
痛点：
  - 负面信息扩散了才发现
  - 缺乏情感分析，不知道舆情严重程度
  - 需要快速生成应对方案
期望：
  - 关键词实时监控 + 秒级告警
  - 情感分析量化舆情严重度
  - AI 生成应对建议
```

#### P2-1：投资 / 金融从业者（决策驱动型）

```
身份：基金经理、券商分析师、量化交易员
场景：热点事件 → 市场影响分析
痛点：
  - 社媒热点对股价的影响难以量化
  - 缺乏舆情因子数据源
期望：
  - 热搜 → 行业/概念股关联
  - 情感指数作为量化因子
  - 历史数据回测支持
```

## 四、下一步开发方向建议

### 4.1 方向矩阵（按价值 × 紧迫度排序）

| 优先级      | 方向            | 目标用户     | 预期价值                     |
| -------- | ------------- | -------- | ------------------------ |
| **P0-A** | 大数据可视化展屏      | 全部用户     | 产品力质的飞跃，从 CLI 工具升级为可视化平台 |
| **P0-B** | 关键词实时告警系统     | 品牌/公关    | 核心商业需求，直接对标万元级 SaaS      |
| **P0-C** | 自动日报/周报生成     | 品牌/数据分析师 | 高频刚需，用户粘性最强的功能           |
| **P1-A** | Web Dashboard | 全部用户     | 降低使用门槛，从开发者扩展到非技术用户      |
| **P1-B** | 历史趋势对比分析      | 数据分析师/投资 | 数据价值深度挖掘                 |
| **P1-C** | 多用户 / 多项目支持   | 团队场景     | 从个人工具升级为团队平台             |
| **P2-A** | 舆情情感指数 API    | 开发者/量化   | 数据产品化，可作为独立 API 服务       |
| **P2-B** | 热点→概念股关联      | 投资从业者    | 垂直场景深挖                   |

### 4.2 重点方向详解

#### P0-A：大数据可视化展屏

**为什么是最优先？** CLI 工具的尽头是可视化。大屏展示是从"工具"升级为"产品"的关键一步。

```
技术方案：
├── 前端：React / Vue3 + ECharts / D3.js
├── 数据层：WebSocket 实时推送 + SQLite/Redis 存储
├── 布局：响应式大屏布局（1920×1080 / 3840×2160）
└── 部署：Docker 一键启动

核心面板：
┌──────────────────────────────────────────────┐
│  🔥 全网热搜实时滚动（5 平台聚合）              │
├──────────────┬───────────────────────────────┤
│  📊 热度趋势  │  🗺️ 地域热力图               │
│  折线图       │  IP 属地分布                   │
├──────────────┼───────────────────────────────┤
│  ☁️ 实时词云  │  📈 平台对比                   │
│  动态更新     │  微博 vs 抖音 vs 知乎          │
├──────────────┼───────────────────────────────┤
│  😊 情感分布  │  🏷️ 热门标签 TOP 20           │
│  正面/负面饼图│  实时排序                      │
├──────────────┴───────────────────────────────┤
│  📋 最新采集内容列表（深度采集结果）             │
└──────────────────────────────────────────────┘

附加能力：
- 定时自动刷新（可配置间隔）
- 全屏 / 聚焦模式
- 截图导出 / 数据下载
- 多主题（暗色/亮色/品牌色）
```

#### P0-B：关键词实时告警系统

```
功能设计：
1. 关键词订阅（品牌词、竞品词、行业词、负面词）
2. 多源实时监控（热搜 + 深度采集 + 搜索）
3. 情感阈值告警（负面比例超 X% 触发）
4. 多通道通知（终端、WebHook、微信/钉钉/飞书）
5. 告警等级（INFO / WARNING / CRITICAL）

技术方案：
- 基于现有 TaskScheduler 扩展
- Redis Pub/Sub 做告警分发
- MQTT 推送告警消息
```

#### P0-C：自动日报/周报生成

```
功能设计：
1. 定时触发（每日 09:00 / 每周一 09:00）
2. AI 自动生成摘要（基于 GLM-5.1）
3. 多格式输出（Markdown / PDF / HTML）
4. 自动推送（邮件 / WebHook / 微信）

报告结构：
- 今日热点 TOP 20
- 各平台热度对比
- 情感分析总览
- 关键词云
- 趋势变化（对比昨日）
- AI 趋势预判
```

#### P1-A：Web Dashboard

```
技术方案：
- 后端：FastAPI / Litestar（Python 生态，复用现有模块）
- 前端：Vue3 + Vite + TailwindCSS + ECharts
- 通信：REST API + WebSocket
- 认证：API Key / OAuth（可选）

核心页面：
- /dashboard    总览大屏
- /hot          热搜聚合
- /deep         深度采集
- /analysis     AI 分析
- /export       数据导出
- /settings     配置管理
- /schedule     定时任务
```

### 4.3 推荐开发路线图

```
Phase 1（4 周）— 可视化 MVP
  ├── P0-A 大数据展屏（React + ECharts）
  ├── 展屏数据 API（FastAPI 后端）
  └── Docker 一键部署

Phase 2（4 周）— 告警 + 报告
  ├── P0-B 关键词实时告警
  ├── P0-C 自动日报/周报
  └── WebHook 通知通道

Phase 3（6 周）— Web 平台
  ├── P1-A Web Dashboard（完整 UI）
  ├── P1-B 历史趋势对比
  └── 多用户基础支持

Phase 4（持续）— 数据产品化
  ├── P2-A 舆情情感指数 API
  ├── P2-B 热点→概念股关联
  └── 开放 API + SDK
```

## 五、商业模式建议

| 层级       | 功能                            | 定价                 | 目标用户    |
| -------- | ----------------------------- | ------------------ | ------- |
| **开源免费** | CLI + 12 数据源采集 + AI 分析        | $0                 | 开发者、极客  |
